Visão geral

Fluxo da documentação e token API

O manual orienta a configuração para geração de token de usuários API dentro do DataCob. O procedimento passa pelo controle de usuários, geração do token, acesso ao Swagger em Ajuda > API Documentação e uso dos endpoints de Account para obter API Key, login e ativar/desativar usuário.

Token por usuárioGerado no controle de usuários, pelo botão Gerar Token API.
Validade máximaO token pode ter validade máxima de 1 ano.
Token anteriorAo gerar novo token, o token anterior é automaticamente revogado.
SwaggerA documentação fica em Ajuda > API Documentação.
Operação

Passo a passo para gerar token API

  1. Acessar o controle de usuários. No DataCob, acesse Operações > Controles > Usuários.
  2. Selecionar o usuário API. Localize o usuário desejado na tela de Controle de Usuários.
  3. Gerar Token API. Clique no botão Gerar Token API para iniciar a geração do token de autenticação.
  4. Definir validade. No campo Novo token para login, informe a data de validade desejada. O limite máximo é de 1 ano.
  5. Atenção ao token anterior. Ao gerar novo token, o token anterior será revogado automaticamente.
  6. Copiar o token gerado. Após a geração, clique em Copiar Token e use o valor nas requisições conforme método de autenticação configurado.
  7. Acessar o Swagger. Para consultar a documentação da API, use Ajuda > API Documentação na tela inicial do DataCob.
Swagger

Endpoints de Account destacados no manual

GET /api/account/v1/obter-apikeyObtém a chave interna usada na autenticação do token API.
POST /api/account/v1/loginCom Login, Password e ApiKey, retorna o token para uso nas demais requisições.
POST /api/account/v1/ativar-desativar-usuarioAtiva ou desativa lista de usuários conforme permissão do usuário autenticado.
Retornos comuns200 OK, 204 Sucesso, 400 dados inválidos, 403 sem permissão.
Checklist

Checklist do analista

  • Confirmar se a solicitação é sobre token API, API Key, Swagger, login ou ativar/desativar usuário.
  • Validar se o usuário utilizado é um usuário API e se possui permissão adequada.
  • Orientar geração do token pelo menu Operações > Controles > Usuários.
  • Confirmar a data de validade desejada, respeitando o limite máximo de 1 ano.
  • Avisar que novo token revoga automaticamente o token anterior.
  • Para documentação, orientar acesso em Ajuda > API Documentação.
  • Em erro 400, validar usuário, senha, API Key e corpo da requisição.
  • Em erro 403, validar permissão do usuário/token e grupo de acesso.
Checklist para o analista: - Confirmar se a solicitação envolve token API, API Key, Swagger, login ou ativar/desativar usuário; - Validar se o usuário é usuário API e possui permissão adequada; - Orientar geração em Operações > Controles > Usuários > Gerar Token API; - Confirmar validade do token, respeitando limite máximo de 1 ano; - Alertar que novo token revoga automaticamente o token anterior; - Para documentação, orientar Ajuda > API Documentação; - Em erro 400, validar usuário, senha, API Key e payload; - Em erro 403, validar permissão do usuário/token e grupo de acesso.
Freshdesk

Resposta sugerida ao cliente

Orientação - Token API DataCob

Olá,

Para geração do token de usuário API no DataCob, acesse:

Operações > Controles > Usuários

Selecione o usuário desejado e clique em "Gerar Token API". Em seguida, informe a validade desejada para o token, respeitando o limite máximo de 1 ano.

Atenção: ao gerar um novo token, o token anterior será automaticamente revogado.

Para consultar as chamadas disponíveis na documentação da API, acesse no DataCob:

Ajuda > API Documentação

Na seção Account, estarão disponíveis as chamadas para obter API Key, realizar login e ativar/desativar usuários, conforme permissão do usuário autenticado.

Caso ocorra erro 400, recomendamos validar usuário, senha, API Key e payload enviado. Em caso de erro 403, validar permissões do usuário/token.
